The Visualisations: Donut Chart

Although there are many various types of monsters that appear in the episodes, we should focus our discussion only on the following 5 types of monsters (Animal, Ghost, Mythical, Possessed Object, and Undead). This is mainly because the appearances other types of monsters are pretty rare, hence the sample size may be inadequate to make a reliable judgement of how each hero fares at tackling these rarer monsters. Besides, this simplifies and scopes the discussion better.

    From Raw Data to CSV file
  1. I removed all the entries from the dataset that had a number of monsters other than one, as I am only interested in the instances where a single monster appeared. This was necessary because the dataset only provides information on whether a character was caught or captured by monsters in general in that episode, and not specifically by which monster. Therefore, I had to filter the data in order to avoid this limitation and allow for a more meaningful discussion.
  2. After filtering out the entries where the monster amount was not equal to one, I generated a pivot table in Excel. In this chart, I grouped the remaining entries by monster type and displayed the count of each monster type. The monster type was placed in the rows section of the pivot chart and the values section displayed the count. I sorted the pivot table according to the count of monsters, and kept only the entries with the top 5 counts.
  3. I then used the Excel Transpose function to have the monster types in the first row, and the corresponding values in the second row. Then, I saved the file as a CSV file and imported it using Fetch API.

The Visualisations: Radar Chart

The purpose of this chart is to provide a high-level overview of how the heroes perform against the top 5 most common types of monsters. In order to achieve this broad perspective, certain details may be lost. Specifically, we will be using a derived value for each character's ability against each monster, calculated by subtracting the number of times the character was captured by the monster from the number of times the character caught the monster (hence losing these two values). The resulting chart will display five polygons, one for each character, with the size of the polygon providing a general indication of the hero's ability level - the larger the polygon, the stronger the hero is considered to be.

    From Raw Data to CSV file
  1. All entries with a number of monsters other than 1 are removed (as with the above donut chart). In addition, all entries with the monster type other than the top 5 common monsters derived above are removed.
  2. For every hero, I carried out the following process
    • I generated a pivot table with the monster type in the row field. Using a binary variable to indicate whether the hero caught the monster, I placed the count of this variable in the values field. To display only the number of times the hero caught the monster, I applied a filter to show only the values that equaled "TRUE".
    • I repeated this to show the number of times the hero was captured by the monster.
    • I combined the above data into one spreadsheet, then derived the value (to be used in the chart, representing the hero’s strength against each monster) by subtracting the number of times the character was captured by the monster from the number of times the character caught the monster.
    • I used the Excel Transpose function to have the monster types in the first row, and the corresponding values of the hero’s strength to that monster in the second row. Then, I saved the file as a CSV file and imported it using Fetch API.

The Visualisations: Horizontal Bar Charts

The aim of these charts is to provide a more detailed examination of how each hero performs against each monster. The values presented in the radar chart were obtained by subtracting the number of times a character is captured by a monster from the number of times the character catches the monster. In contrast, the horizontal bar charts (one for each hero) allow us to show the individual breakdown of these two values: the number of times a hero catches a monster, and the number of times a hero is captured by a monster.

    From Raw Data to CSV file
  1. All entries with a number of monsters other than 1 are removed (as with the above donut chart). In addition, all entries with the monster type other than the top 5 common monsters derived above are removed.
  2. For every hero, I carried out the following process
    • Similar to the process described for creating a radar chart, I started by preparing a spreadsheet that included the number of times each hero caught each type of monster, as well as the number of times each hero was captured by each monster.
    • However, to display this information on the horizontal bar chart, I represented the number of times each hero was captured by a given monster as a negative value. I achieved this by simply adding a negative sign to the value (since there were only five entries).

My progress and errors this week

Navigation Bar

This week, I added a navigation bar that would, in the same webpage, display a different section when the corresponding pages are clicked.

  • In the process of doing this, I encountered new concepts about further splitting my html body into <header>, <nav>, and <main>.
  • While exploring online about how to display the correct section when the navigation bar is clicked on, I encountered the DOM query selector that can select HTML elements and assign them to variables.
  • I faced issues with the styles for the active page not showing up by default (I managed to set the default page to be displayed, but the css styles (100% opacity) and underlines for the selected page were not showing up. To solve this, I had to add the “active” class to the default page using HTML. Thereafter, this “active” class could be removed and added whenever the pages are clicked on.
  • Something I have yet to figure out is how to keep the line under the selected page when I resize the window.

Slider

As I am showing 5 horizontal bar charts, with each pertaining to each character, I decided to put this into a slider.

  • As I explored online, I realised that the JS components of creating this slider is somewhat similar to creating the functional navigation bar, and applied similar concepts, such as adding the ‘active’ CSS class to the selected html element.
  • However, the slider might have been a little more complicated as now there were arrow controls that had to be configured, instead of just clicking on the page the user wants (as in the navigation bar). With the help of Chat GPT, the function called when the event listener receives a click to the left/right arrows make use of +1 and -1 functions to change the ‘currentSlide variable’ (initially set to be 0), to move to the previous and next slides, to keep track of the current slide index and progress forward or backward based on user input.
  • However, one problem I have yet to solve is the disappearance of my chartJS tooltips after I put them into the slider... I’ve tried setting the tooltips display to true using chartJS but it did not work.

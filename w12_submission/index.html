Header Formatting Issues

In the header section, I included both an image and a div that gradually turns black at the lower part. Unfortunately, I faced difficulty in making the image and the div dimensions consistent even when I manually specified their height (while keeping width at 100%). Moreover, the issue intensified when I tried resizing the window.

To address the issue of mismatched sizes between the image and the fade div in my header, ChatGPT recommended using jQuery to dynamically set their heights. The solution involves using the jQuery .height() method to retrieve the image height and applying it to the fade div. To ensure that the function runs both when the page loads and the window is resized, an event listener was added. However, to make the JS portion work, certain modifications had to be made to the CSS code. This includes changing the position settings to relative (some were absolute), removing all height settings, and keeping the width at 100%. Additionally, to maintain the aspect ratio of the header image and allow for auto height, adjustments were made to the height attribute.

Once the issue with the image and fade div heights was resolved, I encountered another problem with the text positioned at the bottom of the header. The text size wasn't adjusting when the window was resized. ChatGPT advised me to use CSS media queries to modify the font size of the text element based on the viewport width. Although this solution was effective, the text sometimes expanded to two lines. To keep the text on a single line, I added the 'white-space' property and set it to nowrap.

Navigation Bar Formatting Issues

Last week, I faced a problem with keeping the line under the selected page when the window is resized.

Following the solution used to fix the issue with the header size, I figured out that a similar approach could be employed here. I decided to add an event listener to the window object that listens for the resize event and updates the width and position of the line accordingly when the event is triggered. This allowed the line to stay in the correct position and size even when the window is resized.

Horizontal Bar Charts: From Chart.JS to Plotly

Initially, I used horizontal bar charts in chart.js to display the positive and negative values of each character's ability, which showed the number of monsters the character caught and the number of times the character was captured by the monster, respectively. However, I encountered a problem when trying to analyze the graphs because certain types of monsters appeared more frequently, making it seem like the characters were stronger as they could catch more of these common monsters, even though it was not necessarily a reflection of their ability. To address this issue, I standardized the numbers to percentages to show the rate at which each character catches the monster. I attempted to change the plot type in chart.js, but it was unsuccessful, so I switched to using plotly instead.

I computed the standardised values required to show the rate at which each character catches the monster. I used EXCEL functions to perform the calculations manually and then entered these percentages into an array.

I used the Plotly guide as a reference for creating my graphs.

I encountered difficulties with the positioning of the legend, but ChatGPT suggested using the yanchor and y properties, which helped me resolve the issue.

I encountered some difficulties with the hover template property and was uncertain about how to personalize my tooltip to display values as percentages. However, chatgpt assisted me with this and it turned out to be a simple problem.

However, despite placing my graphs within the slider like my previous chart.js graphs, I am still encountering issues with tooltips and graph interaction not appearing. This is a matter I will have to address in the upcoming week.

Creating Character Cards

After completing my analysis, my main focus this week was to come up with compelling phrases that accurately highlight the strengths and weaknesses of each character. This required spending considerable time brainstorming and refining the language used to ensure it was both eye-catching and informative.

I started working on creating a card flip effect for an individual card and found inspiration from a YouTube video that can be found at this link. Using the css transform: rotateY() property, I was able to achieve the desired effect.

Additionally, I gained knowledge on JS DOM manipulation through the querySelector feature. By clicking on an element, I was able to modify its class name, allowing relevant CSS properties to be applied. However, when attempting to implement this on all 5 cards, the flip effect only worked on the first card while the remaining 4 were unresponsive. Upon further inspection, I realized that the querySelector only selects the first element with the specified class name. To address this issue, I used querySelectorAll instead and applied a forEach method to add the class name only to the clicked card.

I dedicated most of my time to designing the layout for each of my cards. As I had a lot of div elements involved, it became quite complex, particularly with regards to the alignment, margin, line heights, and padding.
